# WhatHaveIDone (WHID)

**The Modular Personal Data Archeology Suite**

Your digital life is scattered across cloud services you don't control. WHID recovers it — creating local, structured, AI-ready mirrors of your personal data.

## What it does

- **Collects** your data from cloud services into local JSONL vaults
- **Grooms** those vaults — deduplicating, sorting, and validating entries
- **Self-heals** with the Sniper Mechanism: automatically detects missing records and recovers them on the next run

## Architecture

```
WhatHaveIDone/
├── collectors/          # Source-specific data collectors
│   └── gmail_collector.py
├── core/                # Shared processing logic
│   └── groomer.py
├── Vaults/              # Your data lives here (git-ignored)
│   └── Gmail_Primary/
│       ├── 2024/
│       │   ├── 01_January.jsonl
│       │   └── 02_February.jsonl
│       ├── processed_ids.txt
│       └── missing_ids.txt    # (auto-generated by Sniper)
└── config.yaml          # All settings in one place
```

## Quick Start

### Prerequisites

- Python 3.9+
- A Google Cloud project with Gmail API enabled
- OAuth 2.0 credentials (`credentials.json`)

### Setup

```bash
git clone https://github.com/matjes993/WhatHaveIDone.git
cd WhatHaveIDone
pip install -r requirements.txt
```

Place your `credentials.json` in the project root (see [Google's guide](https://developers.google.com/gmail/api/quickstart/python) for setup).

### Collect your Gmail

```bash
python -m collectors.gmail_collector Primary
```

On first run, a browser window opens for OAuth. Your emails are saved to `Vaults/Gmail_Primary/` organized by year and month.

### Groom your vault

```bash
python -m core.groomer Vaults/Gmail_Primary
```

This deduplicates entries, sorts them chronologically, and generates `missing_ids.txt` if any previously processed records are missing from disk.

### Recover missing records (Sniper)

If `missing_ids.txt` exists, the collector automatically enters Sniper mode and recovers only those specific records:

```bash
python -m collectors.gmail_collector Primary
```

## Configuration

Edit `config.yaml` to customize:

```yaml
vault_root: Vaults

gmail:
  max_workers: 3        # Concurrent API threads
  page_size: 500        # Messages per API page
  scope: https://www.googleapis.com/auth/gmail.readonly
  credentials_file: credentials.json
  token_file: token.json
```

## Design Principles

- **Read-only** — your cloud data is never modified (gmail.readonly scope)
- **Append-only collection** — new data is appended, never overwritten during collection
- **Atomic grooming** — vault files are written via temp files to prevent corruption
- **Modular** — each data source gets its own collector; the groomer works with any JSONL vault

## Adding New Collectors

Create a new file in `collectors/`. Each collector should:

1. Authenticate with the source API
2. Fetch records and write them as JSONL to `Vaults/<Source>_<VaultName>/`
3. Track processed IDs in `processed_ids.txt`

The groomer handles deduplication and sorting for any JSONL vault that follows this convention.
